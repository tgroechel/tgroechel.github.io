PoseToCode is a web application where students create code blocks by doing different poses requiring only a computer with a webcam. When the student makes the "run code" pose, the robot executes all the actions from the code block chain. This is a part of a larger research effort from the USC Interaction Lab trying to answer the question "how can we foster curiosity in STEM through kinesthetic learning?". The application was created using Google's Mediapipe for pose detection, Google funded/open-source community supported ml5.js for pose classification, and Google's Blockly for code blocks. PoseToCode took much of its inspiration from the Making a Peanut Butter and Jelly Sandwich exercise given out to new programming students.

What is "block-based" programming?

A common problem when teaching introductory coding is the diffulty of text based coding. The files and build process can be quite complex and unapproachable. Block-based programming takes away those complexities allowing snapping code blocks together to do the same core operations text based code can. This was popularized by MIT's Scratch and has a big presence on code.org. You can use either of these websites for free to start out learning programming!

What is kinesthetic/embodied learning?

Embodied learning (also know as kinesthetic learning) is the study of how we can have students use there bodies/move to learn. There are many educational research projects going on around embodied learning such as at the Cyprus Interaction Lab. A video from their lab is linked below.
